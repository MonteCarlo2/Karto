# Отчетность по блогерам (Google Sheets, на русском)

Этот шаблон рассчитан под правило:
- пользователь пришел по метке блогера;
- закрепление за блогером на 60 дней;
- обновление данных ежедневно.

## 1) Листы в таблице

Создай 4 листа:
- `Справочник_блогеров`
- `Кампании_ручной_ввод`
- `Авто_данные_Supabase`
- `KPI_сводка`

## 2) `Справочник_блогеров`

Колонки:
- `A: Код блогера` (пример: `r7a9k2rta`)
- `B: Имя блогера`
- `C: Канал` (instagram / telegram / youtube)
- `D: Контакт`
- `E: Комментарий`

## 3) `Кампании_ручной_ввод`

Одна строка = одна интеграция.

Колонки:
- `A: Дата старта`
- `B: Дата окончания`
- `C: Код блогера`
- `D: Название кампании`
- `E: Стоимость интеграции, ₽`
- `F: Охват поста` (вручную)
- `G: Охват сторис` (вручную)
- `H: Просмотры публикации (итого)` (вручную)
- `I: Формат` (сторис / рилс / пост / пакет)
- `J: Комментарий`

## 4) `Авто_данные_Supabase`

Сюда ежедневно вставляй выгрузку из view `public.influencer_daily_metrics`:
- `day_moscow`
- `blogger_code`
- `registrations`
- `payments_count`
- `paid_users`
- `revenue_rub`

Можно вставлять через CSV или вручную копировать из Supabase Table Editor.

## 5) `KPI_сводка`

Одна строка = код блогера.

Колонки:
- `A: Код блогера`
- `B: Имя блогера`
- `C: Расход на интеграции, ₽`
- `D: Охват (ручной), итого`
- `E: Регистрации (60 дней)`
- `F: Платежи (60 дней)`
- `G: Платящих пользователей (60 дней)`
- `H: Выручка, ₽ (60 дней)`
- `I: CR охват -> рег`
- `J: CR рег -> платящий`
- `K: CPA, ₽`
- `L: ROAS`
- `M: Прибыль, ₽`

### Рекомендуемые формулы (строка 2)

`C2` (расход):
```gs
=IFERROR(SUMIFS(Кампании_ручной_ввод!$E:$E, Кампании_ручной_ввод!$C:$C, $A2), 0)
```

`D2` (охват):
```gs
=IFERROR(SUMIFS(Кампании_ручной_ввод!$H:$H, Кампании_ручной_ввод!$C:$C, $A2), 0)
```

`E2` (регистрации 60д):
```gs
=IFERROR(SUMIFS(Авто_данные_Supabase!$C:$C, Авто_данные_Supabase!$B:$B, $A2, Авто_данные_Supabase!$A:$A, ">="&TODAY()-60), 0)
```

`F2` (платежи 60д):
```gs
=IFERROR(SUMIFS(Авто_данные_Supabase!$D:$D, Авто_данные_Supabase!$B:$B, $A2, Авто_данные_Supabase!$A:$A, ">="&TODAY()-60), 0)
```

`G2` (платящие 60д):
```gs
=IFERROR(SUMIFS(Авто_данные_Supabase!$E:$E, Авто_данные_Supabase!$B:$B, $A2, Авто_данные_Supabase!$A:$A, ">="&TODAY()-60), 0)
```

`H2` (выручка 60д):
```gs
=IFERROR(SUMIFS(Авто_данные_Supabase!$F:$F, Авто_данные_Supabase!$B:$B, $A2, Авто_данные_Supabase!$A:$A, ">="&TODAY()-60), 0)
```

`I2` (CR охват -> рег):
```gs
=IF(D2=0, 0, E2/D2)
```

`J2` (CR рег -> платящий):
```gs
=IF(E2=0, 0, G2/E2)
```

`K2` (CPA):
```gs
=IF(G2=0, 0, C2/G2)
```

`L2` (ROAS):
```gs
=IF(C2=0, 0, H2/C2)
```

`M2` (Прибыль):
```gs
=H2-C2
```

## 6) Ежедневный регламент (10 минут)

1. Открыть Supabase -> View `public.influencer_daily_metrics`.
2. Скопировать новые строки за текущий день в лист `Авто_данные_Supabase`.
3. В листе `Кампании_ручной_ввод` обновить ручные показатели (охваты, стоимость).
4. В `KPI_сводка` проверить сортировку по `ROAS` и `Прибыль`.

## 7) Что считается автоматически

- Регистрации по коду блогера.
- Платежи, количество платящих, выручка.
- KPI: CR, CPA, ROAS, прибыль.

## 8) Что вносится вручную

- Стоимость интеграции.
- Охваты/просмотры публикаций (если их дает блогер или площадка отдельно).
