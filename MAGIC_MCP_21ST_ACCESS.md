# Доступ к 21st.dev через Magic MCP

## Какой доступ есть у ассистента

У ассистента есть **полный программный доступ** к библиотеке 21st.dev через четыре инструмента Magic MCP:

| Инструмент | Назначение |
|------------|------------|
| **magic_component_builder** | Создание нового UI-компонента по описанию. Возвращает **текстовый сниппет** кода компонента. Поиск по библиотеке 21st.dev (searchQuery), интеграция в указанный файл и проект. |
| **magic_component_inspiration** | Получение **JSON-данных и превью** компонентов с 21st.dev по поисковому запросу — без генерации кода. Для референсов и вдохновения. |
| **magic_component_refiner** | Улучшение/редизайн **уже существующего** React-компонента. Возвращает переработанную версию и инструкции по внедрению. |
| **logo_search** | Поиск логотипов (SVG/JSX/TSX) по названиям брендов. |

Этого достаточно, чтобы собирать интерфейс из 21st.dev: создавать компоненты, смотреть референсы, дорабатывать текущие и подставлять логотипы.

---

## Про открытие браузера

- Ассистент **не вызывает** инструменты Browser MCP (navigate, snapshot, click и т.д.) при запросах к 21st.dev. Все запросы к библиотеке делаются только через Magic MCP (builder, inspiration, refiner, logo_search).
- Если браузер всё равно открывается при работе с Magic MCP, это, скорее всего, делает:
  - **Magic MCP сервер** (например, функция «Real-time Preview» из описания 21st.dev Magic), или
  - **Cursor** (например, превью или OAuth при первом использовании).

**Что можно сделать:**

1. **Проверить настройки Cursor**  
   Настройки → MCP: посмотреть, нет ли у сервера `@21st-dev/magic` опций вроде «Open preview in browser» или «Launch browser».

2. **Конфиг MCP**  
   Файл конфигурации (например, `~/.cursor/mcp.json`). Указан только вызов `npx @21st-dev/magic@latest` и `API_KEY`. Дополнительных переменных для «headless»/«no browser» в официальной документации нет.

3. **Спросить авторов Magic MCP**  
   В [Discord 21st.dev](https://discord.gg/Qx4rFunHfm) или в [Issues на GitHub](https://github.com/21st-dev/magic-mcp/issues) можно уточнить: есть ли способ отключить открытие браузера/превью и работать только через ответы MCP в чате.

---

## Как ассистент будет работать с 21st.dev

- При запросе вида «собери интерфейс с 21st» / «компонент из 21st» ассистент будет использовать **только**:
  - `magic_component_builder` — для генерации и вставки кода;
  - при необходимости `magic_component_inspiration` — для поиска референсов;
  - `magic_component_refiner` — для улучшения существующих компонентов;
  - `logo_search` — для логотипов.
- **Browser MCP вызываться не будет** — никаких navigate/snapshot/click для 21st.dev или превью.

Таким образом, доступ к библиотеке 21st.dev используется по максимуму через MCP, без лишнего открытия браузера со стороны ассистента.
