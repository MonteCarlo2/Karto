# ЮKassa: тестовый магазин (только локально, для скриншотов)

> **Прод (Timeweb):** боевой магазин `1279533` + `live_` ключ — см. [YOOKASSA_PRODUCTION.md](./YOOKASSA_PRODUCTION.md). На сервере менять ничего не нужно, если там уже боевые значения.

Тестовый магазин **Test, karto.pro** — автоплатежи включены по умолчанию. Боевой магазин (`live_…`) для recurring до одобрения менеджером не подходит.

## Ключи (из кабинета)

| Переменная | Значение |
|------------|----------|
| `YOOKASSA_SHOP_ID` | `1280166` |
| `YOOKASSA_SECRET_KEY` | `test_…` — **скопируйте целиком** из кабинета (Интеграция → Ключи API). После вставки: `.\scripts\verify-yookassa.ps1` — должно быть `OK`. |
| `NEXT_PUBLIC_APP_URL` | `http://localhost:3000` (для return после оплаты) |

Боевые ключи сохранены в **`.env.yookassa-prod.snapshot`** (не в git).

## Ошибка «Error in shopId or secret key»

ЮKassa отвечает `invalid_credentials` — ключ не подходит к Shop ID (устарел, опечатка, лишний пробел). Решение:

1. Кабинет → магазин **Test, karto.pro** (Shop ID **1280166**) → **Интеграция → Ключи API**.
2. Скопируйте **Секретный ключ** заново (или **выпустите новый** и скопируйте его).
3. В `.env.local`: `YOOKASSA_SECRET_KEY=test_...` без кавычек и пробелов в конце строки.
4. `.\scripts\verify-yookassa.ps1` → `OK`, затем перезапуск `npm run dev`.

## Переключение

```powershell
# Тестовый магазин
.\scripts\use-yookassa-test.ps1
npm run dev

# Вернуть боевой магазин
.\scripts\use-yookassa-prod.ps1

```

После смены ключей **перезапустите** `npm run dev`.

## Сценарий для скриншотов (отвязка карты)

1. Войти в аккаунт на `http://localhost:3000`.
2. Главная → тариф **«Отзывы»** (например 50 ответов) → в модалке включить **автопродление** → оплатить.
3. Тестовая карта ЮKassa: `5555 5555 5555 4444`, срок — любой будущий, CVC — любой (например `123`). [Документация](https://yookassa.ru/developers/payment-acceptance/testing-and-going-live/testing).
4. После оплаты редирект на `/profile?payment=success` — подтверждение через `/api/payment/confirm` (webhook на localhost не обязателен).
5. В профиле блок **«Автопродление · Отзывы»**:
   - **Выключить** — автопродление off, карта остаётся;
   - **Удалить карту** — подтверждение в диалоге → карта снята с аккаунта.
6. Скрины для ЮKassa: **виден URL** (`localhost:3000/profile` или после выкладки на staging с тест-ключами — `https://karto.pro/profile`), кнопки и диалог «Удалить карту».

## HTTP-уведомления (опционально)

Для локальной разработки достаточно **confirm** при возврате с оплаты. Если нужен webhook: ngrok → `https://….ngrok.io/api/payment/webhook` в кабинете **тестового** магазина.

## Ответ менеджеру ЮKassa

Приложить 2–3 скрина: профиль с URL, диалог «Удалить карту», состояние после удаления («Сохранённая карта» исчезла / «Выключено»).
